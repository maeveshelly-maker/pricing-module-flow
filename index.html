Salesforce Pricing Module — V1 Flow

Products & Price Books, no CPQ · Proposal/RFP stage gate · Updated July 2026

⚑ New to build for V1

Carrier_Association.Preferred_Price_Book__c lookup field · PricebookEntry.Floor__c / Ceiling__c custom fields · a Flow to default the Pricebook · the Proposal/RFP stage-gate validation rule · a Slack-routed Approval Process for out-of-range pricing.

Both new fields are manual, not automated. Preferred_Price_Book__c is set once per carrier by whoever owns that negotiation. PricebookEntry rates are hand-entered from the actual contract terms — nothing here auto-computes a price.

Changing a floor/ceiling or negotiated rate: edit access restricted to Nick/Martha via a permission set, with Field History Tracking on — tracked, not approval-gated. Changes aren't retroactive: a deal already quoted keeps the range it was checked against at the time; the live PricebookEntry is always "current," not versioned per deal.

Flow vs. validation rule: deciding which Pricebook applies has to be a Flow — it sets a value (Opportunity.Pricebook2Id), which a validation rule can't do. Checking whether the entered price is within Floor__c/Ceiling__c is the validation rule's job — a pure block/allow check. Either way, the actual rate + floor + ceiling for a carrier still has to be entered by Nick/Martha — the validation rule enforces against that data, it doesn't replace the need to build it.
